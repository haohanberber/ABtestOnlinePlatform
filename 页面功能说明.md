# 页面功能说明（变更前必读）

> 本文档用于记录当前页面的**全部功能现状**与**开发约定**。  
> 每次新增或修改功能前，请先阅读本文档，再决定实现方案，避免改动冲突和功能回退。

---

## 1. 页面功能总览

当前页面为单页应用（`index.html`），主要包含以下功能区：

1. 顶部标题区  
   - 标题：`AB测试在线平台`  
   - 副标题：`支持实验设计、样本量估算与显著性判断。`

2. 顶部上传区（标题下方）
   - 第一行：`上传分桶ID数据集（用户ID/药店ID）`
   - 第二行：`上传结果数据集（可预览）`

3. 实验设计助手
   - 输入基准转化率、期望转化率、alpha、power、组间比例
   - 支持单双侧切换
   - 输出效果量与最小样本量

4. A/B显著性检验
   - 输入 A/B 访问人数与转化人数（默认空）
   - 输出 z 值、p 值、置信区间、显著性结论
   - 显著性水平 α 不在本模块填写，沿用“实验设计助手”参数
   - 上传结果数据集并完成实验设计后，可自动生成 A/B 检验参数
   - 支持一键“载入案例并计算”（当前内置 `abtest` 案例汇总数据）
   - 支持“是否抽样”切换：全量检验 / 抽样检验
   - 抽样模式下需填写“每组抽样数量”，且不得低于实验设计助手计算的最小样本量（向上取整）
   - 抽样模式下支持填写“抽样 Seed”，同一 Seed 可复现同一抽样结果，变更 Seed 可得到不同结果
   - 首次进入页面时不自动计算，需手动触发（执行检验或载入案例）

5. 在线实验页面入口（新增）
   - 页面左侧新增导航列，当前仅提供 `A/B` 按钮（横向小按钮）
   - `A/B` 按钮位置下移，减少对主页面视觉干扰
   - 默认进入网站时仍展示原有“上传 + 实验设计 + 显著性检验”页面
   - 点击 `A/B` 按钮后切换到“在线实验”独立页面（前端模拟，不连数据库）
   - 在线实验页右上角提供 `返回首页` 按钮，可回到原首页功能区
   - 页面左上角显示小字号层级：列表页为 `Experiment 实验`
   - 列表中实验名称可点击，进入下一层空白详情页
   - 详情页层级为 `Experiment 实验 / 具体实验名`，点击 `Experiment 实验` 可返回上一层列表
   - 点击 `创建实验` 会进入创建页占位，不执行真实创建
   - `创建人` 为人名搜索输入（支持按关键字过滤）
   - `添加筛选项` 为下拉选择，选择后在工具栏下方显示对应筛选控件
   - 实验列表新增 `创建人` 列，预置为：`军儿`、`斯克`、`川普`、`仁勋`、`泰相`
   - 点击实验名进入详情页后，顶部固定展示三个 Tab：`设置`、`结果`、`摘要`
   - `设置` Tab 支持配置：
     - 假说：设置区最上方新增“实验假说”多行文本框
       - 假说内容会同步显示到 `结果` Tab 的“假说板块”
     - 主要指标：可添加/删除，最多 10 个；支持从默认指标下拉选择
       - 默认指标当前包含：`加入购物车`、`DAU(用户)`、`商详页浏览数`
       - 对已添加到主要指标中的默认指标，鼠标悬浮会显示说明卡（`描述`、`解释`、`配置`）
     - 次要指标：可添加/删除，最多 10 个；支持从默认指标下拉选择
     - 持续时间：数字短输入框 + 粒度下拉（`日/周/月`）
     - 分组和参数：
       - `分流`：文案“本次将用于实验的用户ID的百分比为” + 百分比输入框
       - `目标`：定向规则输入框，默认 `所有环境`
       - 分组配置：默认 2 组（`50% 对照组`、`50% 实验组`），支持修改组名、分流比例、描述，支持点击 `+ 新增分组` 添加更多组（最多 10 组）
       - 分组区域采用横向滚动布局，新增分组后可左右滑动查看全部分组卡片与连接箭头
     - 保存设置：设置区新增“保存设置”按钮
       - 点击后将当前实验配置（假说、指标、持续时间、分组参数）保存到浏览器本地
       - 下次重新进入页面并打开对应实验详情时，会自动恢复上次保存的记录
   - `结果` Tab 支持三个板块（从上到下）：
     - 假说板块：实时展示设置区填写的假说
     - 主要指标板块：按“指标名称/分组/用户数/参与率/平均值/标准误差/总计”表格展示，并附结论
     - 次要指标板块：同主要指标板块格式，支持多指标逐项展示并附结论
   - `摘要` Tab 当前为占位区，用于后续扩展

## 2. 上传与预览功能

### 2.1 分桶ID数据集（第一行）

- 上传按钮：`上传分桶ID数据集`
- 预览按钮：`预览`（固定可点击）
- 可上传格式：`.xlsx/.xls/.csv/.tsv/.txt`
- 解析成功后：
  - 自动识别字段，填充 `ID列` 下拉框
  - 默认优先选择类似 `user_id / pharmacy_id / id / 用户id / 药店id` 的字段
  - 可执行分桶与导出结果

### 2.2 结果数据集（第二行）

- 上传按钮：`上传结果数据集`
- 预览按钮：`预览`（固定可点击）
- 可上传格式：`.xlsx/.xls/.csv/.tsv/.txt/.json`
- 仅用于数据预览，不参与分桶

### 2.3 预览交互（两行共用）

- 以居中弹窗展示数据集前10行
- 背景页面置灰/半透明
- 支持右上角 `×` 关闭、点击遮罩关闭、`Esc` 关闭

## 3. 随机分桶功能（第一行）

- 参数项：
  - `分桶数`（2~100，整数）
  - `Seed`（默认 `233`，可手动修改）
  - `ID列`（从上传文件字段中选择）
- 执行逻辑：
  - 对所选 ID 使用 MurmurHash3 128 位哈希
  - `ab_group_id = hash(id) % bucket_count`
  - 当桶数=4：映射为 `A0/A1/A2/B`
  - 其他桶数：命名为 `G0/G1/...`
- 输出能力：
  - 页面展示每组人数统计
  - 下载分桶结果 Excel（新增 `ab_group_id` 与 `ab_group` 列）
- 算法文档：`随机分桶算法.md`

## 4. 当前代码结构约定

### 4.1 运行入口

- 当前页面实际执行入口：`app.js`
- `index.html` 通过普通 `<script src="./app.js"></script>` 引入（兼容直接双击打开 `index.html` 的场景）

### 4.2 模块化目录

- `features/` 目录保留了按功能拆分的模块化实现（用于结构化维护）
- 在线实验页逻辑位于 `features/online-experiment/`
  - `runtime.js`：运行时脚本（通过 `index.html` 直接引入，供 `app.js` 初始化）
  - `index.js`：模块化入口（供 `features/app/main.js` 同步初始化）

## 5. 变更流程（强约定）

每次新增或修改功能，必须按以下顺序执行：

1. 先阅读本文件 `页面功能说明.md`
2. 明确改动影响范围（上传区 / 预览 / 分桶 / 统计 / 实验计算）
3. 再进行代码实现与验证
4. 改动完成后，回写更新本文件（保持文档与代码一致）

---

## 6. 关于 `features` 与 `app.js` 同步

可以，且建议作为固定规范执行。建议规则如下：

- 若改动了 `app.js` 的业务逻辑，需同步到对应 `features/*` 模块
- 若改动了 `features/*` 模块逻辑，需同步回 `app.js`
- 任何一侧更新后，都要做一次“功能对照检查”（上传、预览、分桶、导出、实验计算）

> 说明：当前工程为“兼容优先”（运行走 `app.js`）+“结构化保留”（`features/`）。  
> 因此**不同步会导致两份逻辑漂移**，后续维护风险较高。

补充：
- Cursor 项目规则已固化在 `.cursor/rules/page-change-governance.mdc`
- 项目开发约定见 `开发约定.md`
