# AB测试理论基础

## 1. RCT 与 AB测试的关系

随机对照实验（Randomized Controlled Trial，RCT）是因果推断的金标准。  
AB测试可以理解为 RCT 在互联网产品中的特例：将用户随机分配到不同版本，并比较关键指标差异，以判断改动是否有效。

- A：通常为当前线上版本（Control）
- B：通常为新方案（Treatment）

## 2. 典型业务示例

以电商购物车页优化为例：

- 方案A：购物车页新增用户评价
- 方案B：购物车页新增相似商品推荐

将用户随机分流后，比较两组下单转化率，即可判断哪种方案更优。

## 3. A/B 测试基本流程

1. 明确实验目标和指标：例如验证新落地页是否提升 2% 转化率。  
2. 设计实验方案：定义变量、持续时间、样本量、分组方式。  
3. 选择分析方法：常见方法包括描述性统计、假设检验。  
4. 执行实验并收集数据：按预定方案记录行为数据。  
5. 数据分析及假设检验：通过统计方法判断假设是否成立。  
6. 得出结论及建议：决定是否推广新设计。  

## 4. 实验详细步骤

### 4.1 提出假设

当计划重设计页面并期望提升转化率时，需要先设定统计假设作为实验起点。  
例如，预期新设计可提升 2% 转化率。

可根据业务预期选择检验方式：

- 单尾检验：用于验证“新设计优于旧版”的方向性假设。
- 双尾检验：用于验证“新旧设计存在差异（不限定方向）”。

### 4.2 分组实验

将用户随机分到两组：

- 对照组：继续使用旧版页面。
- 实验组：使用新版页面。

通过埋点记录用户行为（例如是否购买），可将结果编码为：

- `0`：未购买
- `1`：已购买

最终可计算各组平均转化率并进行比较。

### 4.3 计算样本量及实验周期

样本量需要在统计准确性与实验成本之间取得平衡。常见的两比例检验简化样本量公式为：

\[
n = \frac{P_A(1-P_A) + P_B(1-P_B)}{\delta^2} \times \left(Z_{1-\alpha/2} + Z_{1-\beta}\right)^2
\]

其中：

- \(P_A\)：对照组转化率
- \(P_B\)：实验组转化率
- \(\delta = P_B - P_A\)：期望检测效应大小
- \(Z_{1-\alpha/2}\)：双尾检验显著性阈值对应分位数
- \(Z_{1-\beta}\)：统计功效对应分位数

案例参数：

- \(P_A = 13\%\)
- \(P_B = 15\%\)
- \(\alpha = 0.05\)（对应 \(Z_{1-\alpha/2} = 1.96\)）
- \(\beta = 0.2\)（功效 80%，对应 \(Z_{1-\beta} = 0.84\)）

代入可得每组最小样本量约为 `4716`。  
当包含对照组和实验组时，总样本量至少为：

\[
N_{total} = 4716 \times 2 = 9432
\]

若该落地页历史日均浏览量为 `x`，则实验周期下限为：

\[
实验周期(天) = \frac{9432}{x}
\]

### 4.4 实施AB测试并收集数据

通过第一方或第三方实验工具执行分流与埋点，并持续收集用户行为数据。

### 4.5 数据分析与假设检验

在实验达到样本量并结束后，进行指标汇总、显著性检验和区间估计，判断结果是否具有统计意义。

### 4.6 结论与落地建议

基于统计结论评估是否全量发布新方案，或继续迭代并开展下一轮实验。

## 5. A/B测试中的显著性判断

判断两版本差异是否来自真实效果而非随机波动，可采用以下流程：

1. 分别计算两组平均值和标准差（或转化率）。  
2. 计算每组标准误差（SE）。  
3. 使用 t 检验或 z 检验判断差异是否超出随机误差范围。  

若差异显著超出随机误差范围，可认为改动具有统计意义。
